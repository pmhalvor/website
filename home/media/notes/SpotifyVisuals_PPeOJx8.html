Spotify listening history analysis¶

In this article, I'll look into some data I've been pulling from Spotify's API over the past few months. The data is stored as a comma separated file in a FileShare through my Microsoft Azure account. That account also has a Function App currently being called from a Timer Trigger, which sends a request every 3 hours for the last 50 songs played through my Spotify account. (Only new data points are added to the csv.)

The goal of this analysis will be to observe my listening habits, i.e. tops artists and songs, music per time of day, even track meta-data and feature analysis (using the audio-features and audio-analysis calls to Spotify's API). At the end, I'll run a final analysis, where a simple machine learning model will be fit on the data to predict which time of day a specific song may be listened to.

def top(n=25, df=history, show=False):
    artists = df['artist'].value_counts()
    songs = df['name'].value_counts()
    
    if show:
        plt.figure(figsize=(10,int(n/2)))
        artists[:n].plot.barh()
        plt.xlabel('Number of plays')
        plt.title(f'Top {n} artists')
        plt.show()
        
        plt.figure(figsize=(10,int(n/2)))
        songs[:n].plot.barh()
        plt.xlabel('Number of plays')
        plt.title(f'Top {n} songs')
        plt.show()
    return artists[:n],  songs[:n]

def past_months(number_of_months=1):
    then_dt = datetime.now().replace(month=datetime.now().month - number_of_months)
    then = then_dt.strftime('%Y-%m-%dT%H:%M:%S.%f')[:-3]+'Z'
    return history[history['played_at']>then]

Meta-data and Features¶

In order to start analyzing the type of music per day time, we'll need to make some calls to the open Spotify API. Here, I'll read in my client id and secret, again from my environment variables.

In [278]:
import requests as r
import base64

def get_spotify_header():
    # connection string and secret from env. var. 
    client_id = os.getenv('SPOTIFY_CLIENT_ID')
    client_secret = os.getenv('SPOTIFY_CLIENT_SECRET')

    # build header info from env. var. 
    clientb = base64.b64encode(f'{client_id}:{client_secret}'.encode('utf-8'))
    client = str(clientb, 'utf-8')
    header = {'Authorization': f'Basic {client}'}
    token = r.post('https://accounts.spotify.com/api/token', headers=header, data={'grant_type':'client_credentials'}).json()
    header['Authorization'] = 'Bearer ' +token['access_token']
    return header

header = get_spotify_header()

It's important to note that this header is only valid for an hour at a time. It can be refreshed by just calling on get_spotify_header() again.

There are two endpoints we can call here for more details about our songs. audio-features gives a reduced list of interesting metrics of a track, including danceability, energy, loudness, and tempo. audio-analysis gives more practical information about the song's structure, including large codes-strings for metrics like rhythm and tune. We can define functions for these calls for easy use later.

def get_features(track_id=history['id'][0], update_header=False, header=header):
    if update_header:
        header=get_spotify_header() # to make sure this is always updated
    features = r.get(f'https://api.spotify.com/v1/audio-features/{track_id}', headers=header)
    return features.json()

This multi-features request comes in handy if we want these same details for all of the songs in our dataset. There is however a limit of 100 tracks per API call. So if we want the features for all the songs, we'll need to make requests in batches.

def get_full_features(df=history):
    # get unique ids from history df
    history_ids = history['id'].unique()
    
    #create df object to store features
    features_full = pd.DataFrame(columns=features.columns)
    
    # batch calls to api
    for i in tqdm(range(int(len(history_ids)/100)+1)):
        # create start and stop points for batch
        start_idx = i*100
        end_idx = i*100+100
        if end_idx>len(history_ids): # to avoid indexing error
            end_idx = len(history_ids)

        # format id list to string for api
        ids = format_ids(history_ids[start_idx : end_idx])

        # features for ids request
        features_request = r.get(f'https://api.spotify.com/v1/audio-features/?ids={ids}', 
                             headers=header)
        big_features = pd.DataFrame.from_dict(features_request.json()['audio_features'])
        features_full = features_full.append(big_features, ignore_index=True)
    return features_full

Conclusion of our models¶

All the model types we looked at here gave pretty estimates, resulting in high average errors. From this, we can conclude that building a model to correctly a type of song to play during a specific hour of the day is pretty difficult. This is because listening history is a noisy variable, with a number of external influences that are not included in these data.

On average, our models will be roughly 3 hours wrong. However, we saw a doubling of accuracy from a random model, so we do have a good start. For more precise models, I'll have to either add more data, or try different model types. Multilevel models, where different dimensions can be mapped according to different distributions could also be a next step here. The steps will have to be implemented in a later project.

Until then, have fun playing around with this API and maybe tweaking these models. I hope you learned something! :)
